## Midterm Project Title

**KindWords: Daily Motivation Generator for Positivity and Mood Support**

------

## 1. Project Description / Executive Summary

**KindWords** is a simple Android application developed using **Flutter**. Its main purpose is to provide users with quick emotional support through random compliments, motivational quotes, and positive affirmations.

The app is designed for ease of use:

- The user taps one main button to display a random positive message.
- The app includes a built-in collection of **100+ messages**.
- Users can set a **daily notification** at a preferred time.
- Users can **save favorite quotes** and view them later.

The application is built exclusively for **Android** and works **completely offline**, meaning it does not require:

- internet access
- user accounts
- server-side components

The interface is intentionally simple and includes a small animation whenever a new message appears. This keeps the app lightweight, easy to build, and smooth to use.

By the end of the project, the expected deliverables are:

- a working Android prototype
- documentation explaining the project
- source code uploaded to GitHub

This app is intended to be a practical and helpful tool, especially for students who may need a quick mood boost during busy or stressful days.

## 2. Project Objectives

### 2.1 Specific

Develop a simple Android app called **KindWords** using Flutter with the following features:

- a **one-tap button** that shows a random positive message
- a built-in list of **100+ quotes, compliments, and affirmations**
- an option to set **one daily notification time**
- automatic delivery of a random quote through notification
- the ability to **save favorite quotes**
- a separate screen or list to **view saved favorites**

### 2.2 Measurable

The project will be considered measurable and successful if it meets the following:

- contains at least **100 unique quotes**
- displays varied random messages with **no immediate repeats**
- sends the daily notification correctly at the selected time
- keeps saved favorites even after closing and reopening the app
- all features are tested and confirmed working on:
  - an Android emulator
  - a physical Android phone

### 2.3 Achievable

The project is realistic for a midterm because:

- it uses only basic Flutter widgets
- it requires only two lightweight packages:
  - flutter_local_notifications
  - shared_preferences
- it does not require:
  - backend development
  - internet connectivity
  - user authentication
  - complex integrations

The application can be completed within **3 to 5 weeks** using existing skills and available hardware.

### 2.4 Relevant

This project directly supports the goals of the **Application Development and Emerging Technologies** course. It provides hands-on practice in:

- mobile UI design
- Android notification scheduling
- local data storage
- offline-first mobile application development
- basic wellness-focused app design

These are relevant and practical skills aligned with current IT and CS trends in **2026**.

### 2.5 Time-Bound

The following must be completed by the **midterm deadline**:

- a working prototype with all major features connected and tested
- GDD-style or project documentation
- source code uploaded to GitHub
- final project submission

## 4. Project Impact

The **KindWords** app addresses a real and common need among students: having a quick and effortless way to improve their mood during stressful or hectic days.

### User Impact

The app provides value through:

- **one-tap access** to instant positive messages
- **daily reminders** that encourage positive habits
- a **favorites feature** for revisiting meaningful messages
- a simple and low-effort user experience

Because a new random message appears each time, the app also offers a sense of freshness and replay value.

### Technical Impact

From a development perspective, the app demonstrates an efficient and minimalist approach by being fully offline. This avoids common issues such as:

- server costs
- privacy concerns related to account systems
- internet dependency
- network-related failures

### Platform Relevance

The app is focused on **Android**, which is widely used in the Philippines. This makes the project more practical, relevant, and meaningful beyond academic requirements. It also reflects real mobile development skills applicable to local users and real-world use cases.
